# miniRT要件定義

## 開発目的
- 42 Commoncore project として3DCGの基本的な仕組みを実装を通して理解する
- C言語を用いてオブジェクト指向アーキテクチャを実装し、C++等のオブジェクト指向言語がどのようにOOPを実装しているのかを考えるきっかけにする
- チーム開発を通してgithubなど一般的な共同開発ツールの使用に慣れる。

## 業務要件
今回の開発は業務ではないのだが、このプロダクトでできることを記載する。
- CGを用いて写実的に描画された様々なシーンをウィンドウ、またはPPM形式の画像として出力できる。
- 平均1~2分程度の待ち時間でシーンを表示できる
- 独自の形式の入力yファイルを用いてユーザーはあらゆるシーンを作成できる。

## 機能要件
- コマンドライン上で実行ファイルとして起動できる
- コマンドライン引数として.rt拡張子を持つファイルのファイル名を受け取りそこに書かれたテキストをもとにシーンを作成する。
- ウィンドウにシーンを描画する。待ち時間が発生するため、ユーザーに対して残り時間などを表示するような工夫を施すこと
- ウィンドウは適切に操作、削除などできる。プログラムは適切に終了し、再度起動できる
- 詳細は課題PDFに準拠

## 非機能要件
- 一切のメモリリーク、杜撰なメモリ管理を許容しない。
- 待ち時間は、HD720p(1280×720)のウィンドウで、数十オブジェクトが含まれるシーンでも平均1~2分程度にすること。
- 新たなマテリアルや、オブジェクトが追加しやすいような拡張性を担保すること

## その他
- 校舎の環境(ubuntu, clang)を想定している
- ウィンドウへの描画には42の独自ライブラリであるminilibxを用いる
